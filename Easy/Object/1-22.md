1️⃣ BASIC OBJECT ACCESS & PROPERTY OPERATIONS
=====================================================================

PATTERN
-------
Directly access or manipulate object properties.

These operations usually work only on the first level of the object
(no recursion needed).

Problems in this category:
--------------------------

• Count the number of keys in an object
• Get all keys of an object
• Get all values of an object
• Check if an object has a specific key
• Delete a property from an object

Example:

const user = {
  name: "John",
  age: 25
}

Count keys → 2
Keys → ["name","age"]
Values → ["John",25]

Core JavaScript skills used:
----------------------------

Object.keys()
Object.values()
Object.hasOwn()
in operator
delete operator

Category Name:
--------------
Basic Object Property Algorithms

2️⃣ OBJECT VALIDATION & STRUCTURE CHECKING
=====================================================================

PATTERN
-------
Validate the structure or state of an object.

Used heavily in production code to ensure data correctness.

Problems in this category:
--------------------------

• Check if an object is empty
• Check if two objects are equal (shallow)
• Check if the object has nested property
• Count total properties including nested

Example empty object check:

{}

Result → true

Example non-empty:

{ name: "John" }

Result → false

Core JavaScript skills used:
----------------------------

Object.keys().length
for...in
recursion
Object.hasOwn()

Category Name:
--------------
Object Validation Algorithms


=====================================================================
3️⃣ OBJECT COPYING & IMMUTABILITY ALGORITHMS
=====================================================================

PATTERN
-------
Create copies of objects without mutating the original object.

Important in React, Redux, and functional programming.

Problems in this category:
--------------------------

• Clone an object (shallow copy)
• Deep clone an object (JSON method)
• Deep freeze an object
• Merge two objects

Example shallow clone:

const copy = { ...obj }

Example deep clone:

JSON.parse(JSON.stringify(obj))

Core JavaScript skills used:
----------------------------

spread operator (...)
Object.assign()
JSON.parse()
JSON.stringify()
Object.freeze()
recursion

Category Name:
--------------
Object Copy & Immutability Algorithms


=====================================================================
4️⃣ OBJECT TRANSFORMATION ALGORITHMS
=====================================================================

PATTERN
-------
Convert an object from one structure into another structure.

Very common when working with APIs and data formatting.

Problems in this category:
--------------------------

• Convert object to array of key-value pairs
• Convert array of key-value pairs to object
• Invert object keys and values
• Convert object to query string

Example:

Object → Array

{
  name: "John",
  age: 25
}

↓

[
  ["name","John"],
  ["age",25]
]

Example query string:

{
  name: "John",
  age: 25
}

↓

"name=John&age=25"

Core JavaScript skills used:
----------------------------

Object.entries()
Object.fromEntries()
Array.map()
Array.reduce()
encodeURIComponent()

Category Name:
--------------
Object Transformation Algorithms


=====================================================================
5️⃣ OBJECT FILTERING / SELECTION ALGORITHMS
=====================================================================

PATTERN
-------
Select or remove specific properties from an object.

Used frequently when sanitizing API responses or building
clean data structures.

Problems in this category:
--------------------------

• Pick specific keys from an object
• Omit specific keys from an object

Example:

const user = {
  name: "John",
  age: 25,
  password: "123"
}

Pick ["name","age"]

↓

{
  name: "John",
  age: 25
}

Omit ["password"]

↓

{
  name: "John",
  age: 25
}

Core JavaScript skills used:
----------------------------

Object.entries()
filter()
reduce()
Object.fromEntries()

Category Name:
--------------
Object Filtering Algorithms


=====================================================================
6️⃣ DOT NOTATION PATH ALGORITHMS (VERY IMPORTANT)
=====================================================================

PATTERN
-------
Work with nested properties using a string path like:

"user.profile.name"

These algorithms traverse nested objects using a path.

Problems in this category:
--------------------------

• Get value by dot notation string
• Set value by dot notation string
• Check if object has nested property
• Safely destructure nested objects

Example object:

{
  user: {
    profile: {
      name: "John"
    }
  }
}

Path:

"user.profile.name"

Traversal steps:

user → profile → name

These algorithms usually follow the same pattern.

Core JavaScript skills used:
----------------------------

String.split(".")
loop traversal
optional chaining
dynamic property access

Example concept:

const keys = path.split(".")

Then walk through object:

obj[keys[i]]

Category Name:
--------------
Dot Notation Traversal Algorithms


=====================================================================
7️⃣ OBJECT CREATION / SAFE ACCESS PATTERNS
=====================================================================

PATTERN
-------
Safely access or destructure nested values.

These patterns prevent runtime errors when properties
may not exist.

Problems in this category:
--------------------------

• Destructure nested objects safely

Example unsafe:

const name = obj.user.profile.name

If user or profile doesn't exist → error.

Safe version:

const name = obj?.user?.profile?.name ?? "default"

Core JavaScript skills used:
----------------------------

optional chaining (?.)
nullish coalescing (??)
destructuring

Category Name:
--------------
Safe Object Access Algorithms

2️⃣ Transformation

   Verify structure or state of an object

If you master these patterns, you can solve most
JavaScript object algorithm problems used in
